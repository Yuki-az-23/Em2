# Innovation Loop Workflow

Use `/innovation-loop` when starting or continuing work that involves building something new from scratch, learning new technology, R&D, prototyping, or any "figure it out" situation.

## Activation Steps

1. **Read the Full Skill**
   - Read the innovation-loop SKILL.md file at `.agent/innovation-loop/SKILL.md`
   - Understand the three phases and five emotional security rules

2. **Assess Current Phase**
   - Ask the operator: "What are you working on and where are you stuck?"
   - Determine which phase they're in:
     - **Phase 1 (Void)**: Unstructured exploration, breaking things, learning
     - **Phase 2 (Build)**: Assembling scraps into a working prototype
     - **Phase 3 (Extract)**: Documenting methodology and insights

3. **Apply Phase-Appropriate Support**
   - **Phase 1**: Do NOT impose structure. Let chaos flow. Help with research and experimentation.
   - **Phase 2**: Help gather "scraps" (free tools, code snippets, existing solutions). Test ruthlessly.
   - **Phase 3**: Help document WHY things worked. Create reusable frameworks.

4. **Monitor Emotional Security Rules**
   - **Rule 1**: Check time since work began. After 3-4 hours, remind about breaks.
   - **Rule 2**: Acknowledge innovation and impact, not just deliverables.
   - **Rule 3**: No external methodology imposed during Phase 1.
   - **Rule 4**: Treat failures as data. Protect experimentation budget.
   - **Rule 5**: Never let operator skip Phase 3 documentation.

5. **End-of-Session Wrap-Up**
   When operator is done for the day:
   - Summarize what phase they're in
   - Document current state and next steps
   - Celebrate progress (even failed experiments = data)
   - Set intentions for next session

## Quick Commands

- `/innovation-loop status` - Assess current phase and state
- `/innovation-loop wrapup` - End-of-session summary and documentation
- `/innovation-loop break` - Enforce a break reminder with context preservation

## Key Phrases to Listen For

Activate this workflow when hearing:
- "Zero to one" / "0-to-1"
- "Build from scratch"
- "No playbook"
- "Figure it out"
- "Prototype"
- "R&D"
- "Innovation"
- "New product"
- "Learning [new technology]"
- "Unknown territory"
